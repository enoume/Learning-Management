전기기사 자격증 수험생을 위한 **진도 시각화 중심 학습 관리 웹 서비스**

## 1️⃣ 아이디어 및 목적 (Why & What)

### 기획 배경: 이 서비스를 왜 만들게 되었나요?
전기기사 자격증은 과목 수가 많고,  
각 과목 안에서도 **단계(이론·기출 등) → 장 → 강의** 구조가 복잡해  
수험생이 **현재 학습 위치와 전체 진도**를 직관적으로 파악하기 어렵습니다.

기존 학습 방식은 다음과 같은 한계를 가지고 있었습니다.

- 강의 수기 체크 또는 수강 완료 여부 기억 의존
- 엑셀·노션 등 범용 도구 사용 시 진도 계산의 번거로움
- 과목·단계·장 단위로 나뉜 학습 구조의 가시성 부족

이 프로젝트는  
**“지금 어디까지 학습했는지, 남은 학습량이 얼마나 되는지 한눈에 보여주는 것”**을  
핵심 목표로 기획되었습니다.

---

### 주요 기능: 사용자에게 제공하는 핵심 기능
- 과목별 학습 구조 분리 관리 (전기자기학, 전력공학 등)
- 단계(이론/기출/오답 등) → 장 → 강의 단위의 학습 구조 지원
- 강의 완료 체크 기반 전체 진도율 자동 계산
- 완료 강의 수 / 전체 강의 수 실시간 통계 대시보드
- 모달 기반 과목·단계·장·강의 추가 기능
- 로컬 스토리지(localStorage) 기반 데이터 저장
- HTML + CSS + JavaScript만으로 동작하는 경량 웹 앱

---

### 타겟 사용자: 이 앱을 누가 사용하면 좋을까요?
- 전기기사 / 전기산업기사 수험생
- 인강 중심으로 학습하는 독학 수험생
- 학습 “계획”보다 **진도 현황과 가시적 피드백**을 중시하는 사용자
- 설치 없이 가볍게 사용할 수 있는 학습 관리 도구를 원하는 사용자

## 2️⃣ 실제 동작 및 완성도 (How it works)

### 실행 방법
1. 프로젝트 파일 다운로드
2. `index.html` 파일을 웹 브라우저에서 실행
3. 별도 서버, 로그인, 설치 과정 없이 즉시 사용 가능

---

### 핵심 기능 동작 설명

#### 상단 대시보드
- 전체 강의 수 대비 완료 강의 수를 기준으로
  - 전체 진도율(%)
  - 완료 강의 수
  - 누적 학습 시간(분 단위 합산)
  를 실시간으로 계산·표시

#### 좌측 학습 구조 패널
- **과목 선택**
  - 과목별로 학습 데이터를 필터링하여 표시
- **단계 선택**
  - 이론 / 기출 / 오답 등 단계별로 장과 강의를 구분 관리
- **장(Section) 선택**
  - 선택한 단계에 해당하는 장 목록 표시

#### 강의 리스트
- 강의별 체크박스를 통해 완료 여부 관리
- 체크 시
  - 해당 강의 완료 상태 저장
  - 전체 통계 및 진도 즉시 반영
- 강의 삭제는 × 버튼으로 직관적으로 제공

#### 추가 / 수정
- 과목, 단계, 장, 강의 모두 **모달 UI**를 통해 추가
- 상단 타이틀 클릭 시 시험명 수정 가능
  - 다른 시험(예: 건축사, 정보처리기사)에도 재사용 가능

## 3️⃣ AI 협업 및 활용 (AI Collaboration)

### 아이디어 및 구조 설계
- AI와 함께 학습 관리 서비스의 **데이터 구조를 설계**
  - 과목 → 단계 → 장 → 강의의 계층 구조 정의
- 단순 스케줄러가 아닌
  **“진도 시각화 중심 학습 트래커”**로 방향 설정

---

### 구현 과정에서의 AI 활용

#### 문제 1: 강의 체크 시 전체 진도율 계산
- DOM 상태와 로컬 데이터 상태를 분리하여 관리
- 강의 완료 여부를 기준으로
  - 전체 강의 수
  - 완료 강의 수
  를 계산하는 로직을 재설계

**사용한 프롬프트 예시**
강의 체크박스를 클릭했을 때
전체 강의 수 대비 완료 강의 수를 계산하여
상단 진도율과 완료 강의 수를 동시에 갱신하는
JavaScript 로직을 구조적으로 설계해줘


---

#### 문제 2: 과목·단계 전환 시 상태 초기화 문제
- 선택된 과목이나 단계 변경 시
  이전 선택 장이 남아 UI가 꼬이는 문제 발생
- AI와 함께 상태 전이 규칙을 정리하여
  선택 단계 변경 시 장 선택을 초기화하도록 수정

---

### AI 협업을 통해 얻은 점
- AI를 단순 코드 생성기가 아닌 **구조 설계 파트너**로 활용
- 기능 구현 이전에
  - 데이터 구조
  - 상태 흐름
  - UI 반영 순서
  를 먼저 설계하는 습관 형성
- 프론트엔드 상태 관리 개념을 실제 프로젝트에서 체득
